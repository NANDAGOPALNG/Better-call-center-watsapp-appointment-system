# Local Setup

## Prerequisites

- Python 3.12+
- Node.js 20+
- A Supabase project
- Optional Twilio trial account
- Optional Gemini API key

## 1. Database

Run this SQL in the Supabase SQL editor:

```sql
create table if not exists public.appointments (
  id bigint generated by default as identity primary key,
  customer_name text not null,
  phone_number text not null,
  appointment_time timestamptz not null,
  purpose text not null,
  ai_summary text,
  status text not null default 'Scheduled'
    check (status in ('Scheduled', 'Confirmed', 'Reminder Sent', 'Completed', 'Cancelled')),
  confirmation_sent boolean not null default false,
  reminder_sent boolean not null default false,
  whatsapp_confirmation_sent boolean not null default false,
  whatsapp_reminder_sent boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists appointments_time_idx
  on public.appointments (appointment_time);

create index if not exists appointments_reminder_idx
  on public.appointments (reminder_sent, appointment_time);

create table if not exists public.message_logs (
  id bigint generated by default as identity primary key,
  appointment_id bigint not null references public.appointments(id) on delete cascade,
  message_type text not null check (message_type in ('confirmation', 'reminder')),
  channel text not null check (channel in ('mock', 'sms', 'whatsapp')),
  recipient text not null,
  body text not null,
  status text not null check (status in ('sent', 'failed')),
  provider_id text,
  created_at timestamptz not null default now()
);

create index if not exists message_logs_appointment_idx
  on public.message_logs (appointment_id, created_at desc);
```

For an existing appointments table, run:

```sql
alter table public.appointments
  add column if not exists whatsapp_confirmation_sent boolean not null default false,
  add column if not exists whatsapp_reminder_sent boolean not null default false;
```

Restart the backend after applying this migration because schema capability
checks are cached for the life of the API process. WhatsApp-only mode can use
the existing generic flags before migration, but `both` mode needs the separate
columns for accurate partial-delivery retries.

Use a Supabase service-role key only on the backend. Never expose it through a
`VITE_` variable.

## 2. Backend Environment

Copy `backend/.env.example` to `backend/.env`.

| Variable | Required | Purpose |
| --- | --- | --- |
| `SUPABASE_URL` | Yes | Supabase project URL |
| `SUPABASE_KEY` | Yes | Server-side Supabase key |
| `MESSAGING_MODE` | Yes | `mock`, `sms`, `whatsapp`, or `both` |
| `TWILIO_ACCOUNT_SID` | For Twilio | Trial account identifier |
| `TWILIO_AUTH_TOKEN` | For Twilio | Trial account secret |
| `TWILIO_PHONE_NUMBER` | For SMS | Trial SMS sender in E.164 format |
| `TWILIO_WHATSAPP_NUMBER` | For WhatsApp | Sandbox sender in E.164 format |
| `TWILIO_WHATSAPP_ENABLED` | No | Enables or disables WhatsApp |
| `GEMINI_API_KEY` | No | Enables appointment summaries |
| `GEMINI_MODEL` | No | Defaults to `gemini-2.5-flash` |
| `CORS_ORIGINS` | Yes | Comma-separated frontend origins |

Install and run:

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Use `MESSAGING_MODE=mock` for development without consuming Twilio trial units.
Mock messages are logged and, when `message_logs` exists, stored in Supabase.

For free WhatsApp testing:

1. Open Twilio Console's WhatsApp Sandbox.
2. Send the displayed `join ...` message from your WhatsApp account.
3. Set `MESSAGING_MODE=whatsapp`.
4. Set `TWILIO_WHATSAPP_NUMBER` to the Sandbox number.
5. Create appointments using the same joined WhatsApp number.

The API starts without Twilio or Gemini credentials when mock mode is selected.
Without Gemini, the summary becomes `AI summary not configured`.

## 3. Frontend Environment

Copy `frontend/.env.example` to `frontend/.env`:

```dotenv
VITE_API_URL=http://localhost:8000
VITE_AI_ENABLED=true
```

Then run:

```powershell
cd frontend
npm install
npm run dev
```

Set `VITE_AI_ENABLED=false` to skip AI generation requests from the booking
form even when the backend has a Gemini key.

## 4. Verify

- Frontend: `http://localhost:5173`
- API health: `http://localhost:8000/api/health`
- Swagger UI: `http://localhost:8000/docs`
- OpenAPI JSON: `http://localhost:8000/openapi.json`

Create an appointment at least several minutes in the future using an E.164
phone number. Twilio trial SMS can send only to permitted trial recipients, and
WhatsApp Sandbox messages can send only to users who joined that Sandbox.

The Sandbox is appropriate for development. Production business-initiated
WhatsApp reminders generally require an approved WhatsApp sender and approved
content templates.
